### 예시 피드백 (가이드용)

#### ✅ 성공 사례 예시

```markdown
### [2026-01-15] 조직: 프로젝트 지연 문제 해결

**사용한 개념/워크플로우:**
- [[Part1_조직_인력/그래프 이론과 중심성|그래프 이론과 중심성]]
- [[0_Cross_Part_Workflows#워크플로우 1|워크플로우 1: 조직 병목 → 경쟁 우위]]

**초기 상황:**
- 프로젝트가 항상 마감 기한을 놓침
- 모든 사람이 바쁘다고 하지만 진행이 느림
- 특정 팀장에게 일이 집중됨

**적용 과정:**
1. 업무 흐름을 그래프로 시각화 (20분)
2. Betweenness Centrality 계산 → 디자인 팀장 0.85
3. 팀장의 업무 분석 → 50%가 루틴 승인
4. 승인 가이드라인 작성 + 권한 위임

**결과:**
- 프로젝트 처리 시간: 평균 14일 → 7일 (50% 단축)
- 디자인 팀장 업무 부담 40% 감소
- 팀원 자율성 증가, 의사결정 속도 향상

**효과성 평가:** ⭐⭐⭐⭐⭐ (5/5)

**배운 점:**
- 그래프 이론이 직관보다 정확함 (직관: "개발팀이 병목", 실제: "디자인 팀장이 병목")
- 시각화가 경영진 설득에 효과적
- 15분이면 충분히 분석 가능

**개선 제안:**
- 그래프 그리기 도구 추천 추가 (현재 수작업)
- Betweenness Centrality 자동 계산 스크립트 제공

**태그:** #part1 #성공 #조직 #병목제거
```

#### ❌ 실패/블로커 예시

```markdown
### [2026-01-20] 전략: 내쉬 균형 분석 중단

**사용한 개념/워크플로우:**
- [[Part3_전략_경쟁/내쉬 균형|내쉬 균형]]

**초기 상황:**
- 경쟁사와의 가격 경쟁 분석 시도
- 최적 전략 찾고자 함

**적용 과정:**
1. 경쟁 구도 페이오프 매트릭스 작성 시도
2. 경쟁사 전략을 모델링하려 했으나...
   - 경쟁사 정보 부족 (비용 구조 불명)
   - 페이오프를 정량화할 데이터 없음
3. 30분 후 중단

**결과:**
- 분석 실패
- 시간 낭비

**효과성 평가:** ⭐ (1/5)

**배운 점:**
- 내쉬 균형은 데이터가 충분할 때만 가능
- 경쟁사 정보 수집이 선행되어야 함
- 불확실성이 높을 때는 [[Part2_프로세스_운영/베이즈 정리|베이즈 정리]]가 더 적합

**개선 제안:**
- 내쉬 균형 문서에 "사전 요구사항" 섹션 추가
  - [ ] 경쟁사 정보 충분한가?
  - [ ] 페이오프 정량화 가능한가?
  - No라면 → 베이즈 정리 또는 시나리오 플래닝 권장
- [[0_Quick_Start_Guide]]에 "이 도구가 맞지 않는 경우" 추가

**태그:** #part3 #실패 #데이터부족 #개선필요
```

## 불변량 준수 확인

이 문서는 불변량 4 (점진적 개선)을 구현합니다:

| 불변량 | 적용 방식 |
|--------|----------|
| **1. 복잡성 단순화** | 간단한 템플릿, 1-5점 효과성 평가 |
| **2. 데이터 기반** | 실제 적용 사례, 정량적 결과 |
| **3. 구조적 사고** | 반복 패턴 분석, 근본 원인 식별 |
| **4. 점진적 개선** | 피드백 → 개선 제안 → 액션 → 검증 루프 |
| **5. 실용주의** | 성공과 실패 모두 기록, 실행 가능한 개선 |
