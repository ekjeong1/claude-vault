## 📚 학습 철학

> "완벽한 이해가 아닌, 점진적 적용"

- **이론 20% + 실전 80%**: 개념을 배우면 즉시 현실 문제에 적용
- **작은 성공부터**: 큰 프로젝트보다 작은 문제 해결로 시작
- **연결 중심**: 개념 간 관계를 이해하면 기억과 응용이 쉬움
- **반복 학습**: 한 번에 완벽히 이해하려 하지 말고 여러 번 반복

## 🛤️ 3가지 학습 경로

### 경로 1: 문제 해결 중심 (빠른 적용)

**대상:** 당장 해결할 문제가 있는 실무자

**특징:**
- 필요한 개념만 선택 학습
- 즉시 실전 적용
- 빠른 성과 도출

**단계:**

```
1단계: 문제 정의 (1일)
   └─ 현재 가장 큰 문제가 무엇인가?
   └─ [[0_Quick_Reference]] 에서 관련 개념 찾기

2단계: 개념 학습 (2-3일)
   └─ 해당 개념 노트 읽기
   └─ 예시와 적용 방법 이해
   └─ [[0_Glossary]] 에서 용어 확인

3단계: 실전 적용 (1주)
   └─ 실제 데이터로 분석
   └─ 작은 범위부터 시작
   └─ 결과 문서화

4단계: 확장 (지속)
   └─ 관련 개념 추가 학습
   └─ 다른 문제에 적용
   └─ [[0_Concept_Map]] 에서 연결 탐색
```

**추천 첫 개념:**
- 병목 문제 → [[Part1_조직_인력/그래프 이론과 중심성|그래프 이론]]

## 💡 학습 팁

### 효과적인 학습 방법

**1. 능동적 읽기**
- 읽으면서 질문하기: "우리 조직에서는?"
- 예시 만들기: 실제 상황에 대입
- 노트 작성: 핵심을 자기 언어로

**2. 즉시 적용**
- 읽은 후 24시간 내 적용
- 작은 것부터: 완벽한 분석보다 빠른 시도
- 실패 환영: 실패에서 더 많이 배움

**3. 연결 만들기**
- [[0_Concept_Map]] 자주 보기
- 새 개념과 기존 지식 연결
- Part 간 유사점/차이점 비교

**4. 반복 노출**
- 한 번에 완벽히 이해 불가
- 같은 개념을 다른 맥락에서 재학습
- 2-3개월 후 복습

### 피해야 할 것

**❌ 수동적 읽기만**
- 읽기만 하면 기억 안 됨
- 반드시 실습 동반

**❌ 완벽주의**
- 80% 이해면 충분
- 적용하면서 나머지 20% 채워짐

**❌ 순서 무시**
- 어려운 Part부터 시작하면 좌절
- 추천 순서 따르기

**❌ 고립 학습**
- 혼자보다 토론이 효과적
- 동료와 학습 그룹 만들기
