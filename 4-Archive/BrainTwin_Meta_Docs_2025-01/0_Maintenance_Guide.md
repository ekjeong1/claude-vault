## 📂 파일 네이밍 규칙

### 인덱스 파일

```
0_[이름]_Index.md
```

**규칙:**
- `0_`로 시작 (폴더 최상단 정렬)
- Title Case, 단어 간 `_`
- `_Index` 접미사
- 영문 사용 권장

**예시:**
- ✅ `0_Part1_Organization_Index.md`
- ✅ `0_Resources_Index.md`
- ❌ `part1-index.md`
- ❌ `조직_인덱스.md`

### 개념 파일

```
[개념명].md
```

**규칙:**
- 명확하고 간결한 이름
- 한글/영문 모두 가능
- 공백 사용 가능
- 특수문자 피하기 (`/`, `\`, `:`, `*`, `?`, `"`, `<`, `>`, `|`)

**예시:**
- ✅ `그래프 이론과 중심성.md`
- ✅ `SVD와 PCA.md`
- ✅ `Phase Transition.md`
- ❌ `그래프/이론.md` (슬래시 사용 금지)

### 보조 파일

```
[목적]_[타입].md
```

**예시:**
- `Organizational_Analysis_Template.md`
- `Organizational_Math_Case_Studies.md`

## 🔗 링크 규칙

### Wiki 링크 형식

**기본 형식:**
```markdown
[[파일명]]
[[파일명|표시 텍스트]]
```

**섹션 링크:**
```markdown
[[파일명#섹션명]]
[[파일명#섹션명|표시 텍스트]]
```

**상대 경로:**
```markdown
[[../폴더/파일명]]
[[0_Index_Name]]  # 같은 폴더
```

### 링크 원칙

1. **일관성**: 같은 파일은 같은 방식으로 링크
2. **명확성**: 링크 텍스트가 목적지를 명확히 표현
3. **유지보수**: 파일명 변경 시 모든 링크 업데이트
4. **깨진 링크 없음**: Obsidian에서 정기적으로 확인

## 🚨 일반적인 문제와 해결

### 문제: 파일명 변경 시 링크가 깨짐

**해결:**
1. Obsidian에서 파일명 변경 (자동 링크 업데이트)
2. 또는 수동으로 모든 참조 검색 후 업데이트
3. Git으로 `git mv` 사용하여 이력 보존

### 문제: 중복된 개념

**해결:**
1. 더 포괄적인 개념으로 통합
2. 한 파일을 다른 파일로 리다이렉트
3. 명확히 구분하여 각각의 관점 설명

### 문제: Part 간 경계가 모호

**해결:**
1. 개념의 주요 초점으로 분류
2. 다중 분류보다는 주 분류 + 크로스 레퍼런스
3. 개념 관계도에서 연결 명확히 표시
