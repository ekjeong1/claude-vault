# 베이즈 정리

데이터가 들어올 때마다 믿음을 업데이트하는 확률론적 사고.

## 핵심 개념

**베이즈 정리**는 사전 확률(prior)을 새로운 증거(likelihood)로 갱신해 사후 확률(posterior)을 계산한다.

```
P(가설|데이터) = P(데이터|가설) × P(가설) / P(데이터)
```

## 수학적 정의

**베이즈 정리(Bayes' Theorem)**:

$$P(A|B) = \frac{P(B|A) \cdot P(A)}{P(B)}$$

여기서:
- $P(A|B)$: 사건 $B$가 발생했을 때 $A$의 사후확률(Posterior)
- $P(B|A)$: 사건 $A$가 발생했을 때 $B$의 우도(Likelihood)
- $P(A)$: 사건 $A$의 사전확률(Prior)
- $P(B)$: 사건 $B$의 주변확률(Marginal)

**연속 형태**:
$$f(x|y) = \frac{f(y|x) \cdot f(x)}{f(y)} = \frac{f(y|x) \cdot f(x)}{\int f(y|x) f(x) dx}$$
## 비즈니스 적용

- A/B 테스트 해석: 초기 가설을 데이터로 점진적 업데이트
- 수요 예측: 과거 패턴에 최신 트렌드 반영
- 리스크 관리: 사고 확률을 새로운 신호로 조정

## 예시

신제품 성공률 추정치 30% → 초기 판매 데이터 유입 → 베이즈 갱신으로 45%로 상향

## 효과

불확실성 하에서도 합리적 의사결정, 과신 방지, 학습하는 전략

## 한계점

- **사전 확률의 주관성**: 초기 가정이 자의적일 수 있음
- **잘못된 사전 확률의 영향**: 데이터가 충분하지 않으면 편향된 결론 가능
- **적용 시 주의**: 사전 확률 설정 근거를 명확히 하고, 충분한 데이터로 갱신 필요
