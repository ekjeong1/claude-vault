# 조직 분석 시각화 가이드

수학적 프레임워크를 시각적으로 표현하는 방법론.

[[0_Part1_Organization_Index|조직 분석 수학]]의 세 가지 개념을 다이어그램으로 만들어 직관적으로 이해한다.

---

## 1. 그래프 이론: 네트워크 다이어그램

### 기본 표기법

```
[노드] = 사람, 팀, 시스템
─────> = 방향성 있는 연결 (의사결정, 정보 흐름)
────── = 방향성 없는 연결 (협업, 소통)
(굵기) = 연결 강도 (빈도, 중요도)
```

### 예시: 스타트업 의사결정 네트워크

```
        [CEO]
       /  |  \
      /   |   \
   [CTO] [COO] [CMO]
    / \    |    / \
   /   \   |   /   \
[Dev1][Dev2][PM][Mkt1][Mkt2]
```

**중심성 평가**:
- **CEO**: Betweenness 높음 (모든 경로가 통과) → 병목
- **PM**: Betweeness 중간 (Dev-Mkt 연결) → 조정자
- **Dev1, Mkt1**: Degree 낮음 → 말단

### 병목 시각화

**Before (병목 있음)**:
```
      [Manager]  ← 모든 결정이 집중
     /  |  |  \
    /   |  |   \
 [A]  [B] [C]  [D]
```

**After (권한 위임)**:
```
  [Manager]
     |   |
  [Lead1][Lead2]  ← 중간 레이어 추가
   /  \    /  \
 [A]  [B][C]  [D]
```

### 실습 템플릿

1. **노드 나열**: 주요 의사결정자 5-10명
2. **엣지 그리기**: 승인/보고 관계 표시
3. **가중치 추가**: 빈도(일일/주간/월간)
4. **병목 표시**: 가장 많은 경로가 지나는 노드 강조

## 2. Rank-Nullity: 가치 매트릭스

### 2x2 매트릭스 표현

```
    독립적 기여도
        ^
  High  |
        |  [A팀]     [B팀]
        |  (Rank)    (Rank)
  ------+-------------------->
        |           [C팀]
  Low   |  [D팀]    (Nullity)
        |  (Nullity)
        |
    중복/불필요
```

**해석**:
- **A, B팀**: 고유한 가치 창출 → 유지/강화
- **C팀**: 독립적이지만 중복 많음 → 정리 필요
- **D팀**: 가치 낮고 중복 높음 → 재배치/제거

### 가치 스택 다이어그램

```
전체 업무량
┌─────────────┐
│   Nullity   │ ← 중복/불필요 업무 (제거 대상)
├─────────────┤
│             │
│    Rank     │ ← 독립적 가치 창출 (핵심)
│             │
└─────────────┘
```

**목표**: Rank 증가, Nullity 감소

### 업무 분해 차트

```
팀의 총 업무 시간 (100%)
├── 핵심 고유 업무 (Rank): 40%
│   ├── 제품 개발: 25%
│   └── 고객 지원: 15%
└── 중복/불필요 업무 (Nullity): 60%
    ├── 중복 보고서: 20%
    ├── 불필요한 회의: 25%
    └── 수작업 가능 자동화: 15%
```

### 실습 템플릿

1. **팀/개인별 주요 활동 나열**
2. **독립성 점수 부여** (1-10)
3. **중복도 점수 부여** (1-10)
4. **매트릭스에 플롯**
5. **각 사분면별 액션 정의**

## 3. 군론: 패턴 타임라인

### 반복 패턴 시각화

```
2024-Q1      2024-Q2      2024-Q3      2024-Q4
   |            |            |            |
 [문제A]      [문제A']    [문제A'']    [문제A''']
   ↓            ↓            ↓            ↓
 표면: 배포    표면: 품질   표면: 일정   표면: 리소스
 지연          이슈         지연         부족
   ↓            ↓            ↓            ↓
 불변량: 테스트 자동화 부재 (구조적 원인)
```

**해석**: 표면적 증상은 다르지만, 본질은 같은 문제

### 대칭성 매핑

```
팀 A                팀 B
  |                   |
[같은 문제]        [같은 문제]
  ↓                   ↓
커뮤니케이션       커뮤니케이션
 단절                단절
  ↓                   ↓
동형 구조: 조직 사일로 (Organizational Silos)
```

**구조적 원인**: 팀 간 정보 공유 부재

### 문제 분류 트리

```
반복 문제들
├── 카테고리 1: 의사결정 지연
│   ├── 프로젝트 A에서 발생
│   ├── 프로젝트 B에서 발생
│   └── 불변량: 승인 권한 불명확
├── 카테고리 2: 리소스 부족
│   ├── 팀 A에서 발생
│   ├── 팀 B에서 발생
│   └── 불변량: 우선순위 정책 부재
└── 카테고리 3: 품질 이슈
    ├── Q1에서 발생
    ├── Q3에서 발생
    └── 불변량: 테스트 인프라 미비
```

### 실습 템플릿

1. **최근 6개월 문제 나열**
2. **발생 시점 타임라인 작성**
3. **표면적 증상 vs 본질 구분**
4. **불변량(공통 요소) 찾기**
5. **구조적 원인 가설 수립**

## 통합 시각화: 3단계 대시보드

### 종합 진단 보드

```
┌─────────────────────────────────────────┐
│ 1단계: 네트워크 (그래프 이론)            │
│                                         │
│      [CEO]  ← 병목 #1                   │
│     /  |  \                             │
│   [X] [Y] [Z]                           │
│                                         │
├─────────────────────────────────────────┤
│ 2단계: 가치 매트릭스 (Rank-Nullity)      │
│                                         │
│   Rank: 40%  ████░░░░░░                 │
│   Nullity: 60%  ████████░░              │
│   → 중복 업무 제거 필요                  │
│                                         │
├─────────────────────────────────────────┤
│ 3단계: 패턴 (군론)                       │
│                                         │
│   Q1 ─→ Q2 ─→ Q3 ─→ Q4                 │
│   [A]   [A']  [A''] [A''']              │
│                                         │
│   불변량: 테스트 자동화 부재              │
│                                         │
└─────────────────────────────────────────┘

종합 인사이트: CEO 병목 + 중복 업무 60% +
구조적 문제(테스트 인프라)
```

## 도구 추천

### 디지털 도구
- **네트워크 다이어그램**: Miro, Figma, draw.io
- **매트릭스**: Google Sheets, Excel, Notion
- **타임라인**: Miro, Lucidchart, Timeline.js

### 아날로그 도구
- **화이트보드**: 초기 브레인스토밍
- **포스트잇**: 노드/요소 이동 가능
- **색깔 펜**: 중요도/카테고리 구분
