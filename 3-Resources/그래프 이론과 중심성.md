# 그래프 이론과 중심성

병목 지점과 권한 위임

## 개요

그래프 이론의 중심성(Centrality) 개념을 조직 구조와 의사결정 흐름 분석에 활용.

## 중심성 지표

- **Degree Centrality**: 직접 연결 수
- **Betweenness Centrality**: 경로 사이에 위치하는 정도 (병목 지점)
- **Closeness Centrality**: 전체와의 평균 거리
- **Eigenvector Centrality**: 영향력 있는 노드와의 연결

## 적용

조직 내 병목 지점을 찾아내고, 효율적인 권한 위임과 커뮤니케이션 구조 설계에 활용.

병목 지점으로 식별된 노드의 실질적 기여도는 [[Rank와 Nullity]]로 측정할 수 있다. 같은 병목이 반복된다면 [[군론과 대칭성]]으로 패턴의 근본 원인을 분석한다.

## 실전 적용 방법

### 데이터 수집

**관찰 기간**: 최소 1주일

**수집할 정보**:
- 의사결정 흐름: 누가 누구에게 승인을 받는가?
- 커뮤니케이션 패턴: 정보가 어떻게 전달되는가?
- 대기 시간: 어느 단계에서 가장 오래 걸리는가?

**방법**:
- 회의록/메신저 로그 검토
- 업무 플로우 다이어그램 작성
- 팀원 인터뷰 (누구와 가장 자주 소통하는가?)

### 분석 절차

1. **네트워크 다이어그램 작성**
   - 노드: 사람, 팀, 시스템
   - 엣지: 커뮤니케이션/의사결정 연결
   - 가중치: 빈도나 중요도

2. **중심성 계산** (정성적 평가 가능)
   - Betweenness: 이 사람 없이 일이 진행될 수 있는가?
   - Degree: 몇 명과 직접 연결되어 있는가?
   - Closeness: 정보가 이 사람에게 얼마나 빨리 도달하는가?

3. **병목 지점 식별**
   - 높은 Betweenness + 낮은 처리 속도
   - 한 사람에게 과도하게 집중된 의사결정
   - 부재 시 업무가 멈추는 지점

### 질문 체크리스트

- [ ] 특정 사람이 휴가/부재 시 업무가 멈추는가?
- [ ] 모든 승인이 한 사람을 거쳐가는가?
- [ ] 정보 전달이 특정 구간에서 지연되는가?
- [ ] 중요한 결정이 소수에게만 집중되어 있는가?

### 산출물

**병목 지점 리스트**:
- 누가 병목인가?
- 왜 병목이 되는가? (권한, 전문성, 정보 독점)
- 대안은 무엇인가?

**권한 위임 계획**:
- 어떤 결정을 누구에게 위임할 것인가?
- 필요한 정보/권한 이전은?
- 백업 체계는?
