## 📖 개요

이 가이드는 BrainTwin에 새로운 개념을 추가하거나 기존 개념을 검토할 때 **5가지 불변량 준수**를 확인하는 통합 문서입니다.

**핵심 문서:**
- 불변량 정의: [[0_Invariants]] (변경 불가, 참조만)
- 이 문서: 검증 프로세스와 체크리스트
- 정기 검증: 분기별 1회

---

## 🎯 5가지 불변량 (요약)

전체 내용은 [[0_Invariants|불변량 정의 문서]] 참조

| 불변량 | 핵심 원칙 | 질문 |
|--------|----------|------|
| **1. 복잡성 단순화** | 본질만 남기고 제거 | 이것이 복잡성을 줄이는가? |
| **2. 데이터 기반** | 직관이 아닌 수학적 근거 | 측정 가능하고 검증 가능한가? |
| **3. 구조적 사고** | 표면이 아닌 근본 원인 | 근본 원인을 해결하는가? |
| **4. 점진적 개선** | 완벽이 아닌 학습하는 시스템 | 학습하고 개선되는가? |
| **5. 실용주의** | 이론이 아닌 비즈니스 문제 해결 | 비즈니스 가치를 창출하는가? |

## ✅ 신규 개념 검증 체크리스트

### 빠른 검증 (5분)

**최소 요구사항:**
- [ ] 복잡성을 줄이는가?
- [ ] 데이터로 측정 가능한가?
- [ ] 구조적 원인을 해결하는가?
- [ ] 학습하고 개선되는가?
- [ ] 비즈니스 가치가 있는가?

**통과 기준:** 5개 중 최소 4개 Yes

---

### 상세 검증 (20-30분)

#### 불변량 1: 복잡성 단순화 (5점 만점)

**Q1. 압축성**
- 5점: 10개 이상 변수 → 2-3개 핵심 요인
- 3점: 복잡성 유지
- 0점: 복잡성 증가

**Q2. 직관성**
- 5점: 수학 배경 없이 즉시 이해
- 3점: 기초 수학, 20-30분 학습
- 0점: 수학이 목적이 됨

**Q3. 즉시성**
- 5점: 별도 학습 없이 바로 사용
- 3점: 1시간 내 준비 후 적용
- 0점: 전문가 도움 필수

**Q4. 필요성**
- 5점: 유일한 솔루션
- 3점: 대안보다 2-3배 효율적
- 0점: 불필요

**소계:** ___/20

---

#### 불변량 2: 데이터 기반 객관성 (5점 만점)

**Q5. 측정 가능성**
- 5점: 명확한 숫자/지표, 자동 측정
- 3점: 반정량적 (순위, 등급)
- 0점: 측정 불가능

**Q6. 재현성**
- 5점: 완전히 재현 가능, 자동화 가능
- 3점: 대체로 재현 가능
- 0점: 재현 불가능

**Q7. 검증 가능성**
- 5점: A/B 테스트, 실험 설계 가능
- 3점: 간접적 검증 가능
- 0점: 검증 불가능

**Q8. 편향 제거**
- 5점: 편향을 체계적으로 제거
- 3점: 일부 편향 존재하나 인지
- 0점: HiPPO에 좌우

**소계:** ___/20

---

#### 불변량 3: 구조적 사고 (5점 만점)

**Q9. 반복성 분석**
- 5점: 반복 문제의 구조적 원인 식별
- 3점: 증상과 원인 구분
- 0점: 증상만 치료

**Q10. 피드백 식별**
- 5점: 강화/균형 루프 명확히 시각화
- 3점: 인과관계 이해
- 0점: 피드백 개념 없음

**Q11. 레버리지 발견**
- 5점: 레버리지 포인트 명확, 실행 계획
- 3점: 중요한 요인 파악
- 0점: 전체 시스템 무시

**Q12. 지속 가능성**
- 5점: 구조 변경으로 재발 차단
- 3점: 재발 가능성 일부 존재
- 0점: 문제 악화 가능

**소계:** ___/20

---

#### 불변량 4: 점진적 개선 (5점 만점)

**Q13. 업데이트 가능성**
- 5점: 실시간 자동 업데이트
- 3점: 분기 단위 수동 업데이트
- 0점: 정적 모델

**Q14. 실험 설계**
- 5점: 실시간 A/B, Multi-Armed Bandit
- 3점: 월 단위 실험 가능
- 0점: 완벽주의 (실험 없음)

**Q15. 피드백 속도**
- 5점: 실시간 피드백
- 3점: 주 단위 피드백
- 0점: 피드백 없음

**Q16. 실패 허용**
- 5점: 실패에서 체계적 학습
- 3점: 실패 기록
- 0점: 완벽한 모델 요구

**소계:** ___/20

---

#### 불변량 5: 실용주의 (5점 만점)

**Q17. 비즈니스 문제 정의**
- 5점: 명확한 비즈니스 문제, 측정 가능
- 3점: 비즈니스 연결 가능
- 0점: 수학이 목적

**Q18. ROI 검증**
- 5점: ROI > 500%
- 3점: ROI > 100%
- 0점: ROI 측정 안 함

**Q19. 실행 가능성**
- 5점: 즉시 실행, 실제 의사결정 사용
- 3점: 일부 실행 가능
- 0점: 솔루션이 문제를 찾음

**Q20. 이해관계자 수용**
- 5점: 경영진과 실무자 모두 수용
- 3점: 일부 수용
- 0점: 수용 불가능

**소계:** ___/20

## 📊 불변량 위반 사례와 올바른 예시

### 위반 사례 1: 복잡성 증가 ❌

**잘못된 예: 5단계 분석 프레임워크**
```
1. 데이터 수집 (20개 지표)
2. 상관관계 매트릭스
3. 인자 분석
4. 다차원 회귀
5. 시뮬레이션
→ 문제: 복잡도가 오히려 증가
```

**올바른 예: SVD/PCA** ✅
```
문제: 30개 KPI 파악 어려움
솔루션: SVD로 2-3개 요인 압축
결과: 한눈에 상황 파악
```

---

### 위반 사례 2: 수학이 목적 ❌

**잘못된 예: 수학적 엄밀성 강조**
```
"Hilbert 공간의 orthogonal projection을 사용하여
다차원 metric space에서 optimal subspace를 찾습니다"
→ 문제: 비즈니스 의미 불명확
```

**올바른 예: 그래프 이론** ✅
```
문제: 조직 병목이 어디인지 모름
솔루션: 업무 흐름을 그래프로 그리고 중심성 계산
결과: 병목 노드 시각적으로 즉시 식별
```

---

### 위반 사례 3: 주관적 판단 ❌

**잘못된 예: 직관 기반**
```
"경험상 이 전략이 효과적일 것 같습니다"
→ 문제: 측정 불가, 검증 불가
```

**올바른 예: 베이즈 업데이트** ✅
```
초기 가설: 성공률 50%
데이터 수집: 시장 테스트 100회
베이즈 업데이트: P(성공|데이터) = 68% (95% CI: 61-75%)
결정: 데이터 기반, 불확실성 정량화
```

---

### 위반 사례 4: 증상만 치료 ❌

**잘못된 예: 임시방편**
```
문제: 매달 마감 기한 놓침
잘못된 해결책: 야근으로 해결
→ 문제: 다음 달 반복
```

**올바른 예: 구조 변경** ✅
```
반복 문제: 프로젝트 항상 지연
구조 분석: 그래프 이론으로 병목 식별
발견: 디자인 팀장 1명이 모든 승인 (중심성 최고)
해결책: 승인 권한 위임, 가이드라인 작성
결과: 병목 제거, 재발 방지
```

---

### 위반 사례 5: 정적 시스템 ❌

**잘못된 예: 일회성 분석**
```
"조직 구조 분석 완료. 보고서 제출."
→ 3개월 후 조직 변경, 분석 무용지물
```

**올바른 예: Multi-Armed Bandit** ✅
```
상황: 3개 마케팅 캠페인 중 최선은?
BrainTwin: Multi-Armed Bandit
- 처음 균등 배분
- 성과 좋은 캠페인에 점진적으로 더 많은 트래픽
- 실시간 학습 및 최적화
결과: 기회비용 최소화, 지속적 개선
```

## 🔄 정기 검증 프로세스

### 분기별 검증 (분기 말)

**1단계: 데이터 수집**
- 사용 빈도 분석
- 피드백 수집
- 효과성 평가

**2단계: 불변량 체크**
- 모든 개념에 대해 5가지 불변량 평가 (0-5점)
- 총점 계산 (0-25점)

**3단계: 의사결정**
```
점수 ≥ 20 → ✅ 유지 및 강화
15 ≤ 점수 < 20 → ⚠️ 개선 계획 (1개월)
10 ≤ 점수 < 15 → 🔴 재설계 또는 Archive
점수 < 10 → ❌ 즉시 Archive
```

**4단계: 액션 실행**
- 개선 필요: 1개월 내 개선
- 재설계 필요: Archive 이동 또는 재작성
- 제거: 4-Archive로 이동

## 📝 신규 개념 추가 가이드

### 추가 프로세스

**1. 개념 초안 작성**
- 비즈니스 문제 정의
- 수학적 접근법
- 실전 사례
- 워크시트/템플릿

**2. 불변량 체크리스트 작성**
- 위의 20개 질문에 답변
- 총점 계산
- 80점 이상 필요

**3. 피어 리뷰**
- 다른 사람이 이해 가능한지 확인
- 실제 적용 가능한지 테스트

**4. 통합**
- 해당 Part 인덱스에 추가
- Master Index 업데이트
- 관련 개념과 링크 연결

**5. 검증 완료 표시**
- 개념 파일 상단에 "✅ 불변량 검증 완료 (YYYY-MM-DD)" 추가
