## 📝 피드백 수집 및 처리

### 피드백 수집 채널

**내부:**
- 사용자 인터뷰
- 워크샵 후 설문
- 실제 적용 사례

**외부 (선택):**
- GitHub Issues
- 이메일

---

### 피드백 기록 템플릿

| 날짜 | 개념/Part | 피드백 내용 | 유형 | 우선순위 | 상태 |
|------|-----------|-------------|------|----------|------|
| 2026-01-10 | SVD/PCA | "이해하기 쉬웠음" | 긍정 | - | 완료 |
| 2026-01-15 | Part 6 | "너무 어려움" | 개선 | 높음 | 진행중 |
| 2026-01-20 | 그래프 이론 | "실제 적용 성공" | 사례 | - | 완료 |

**유형 분류:**
- **긍정**: 잘 작동, 효과적
- **개선**: 개선 필요, 불명확
- **버그**: 링크 깨짐, 오류
- **사례**: 실제 적용 성공/실패

**우선순위:**
- **높음**: 즉시 처리 (1주 내)
- **중간**: 다음 월간 작업 (1개월 내)
- **낮음**: 분기별 검토 (3개월 내)

---

### 피드백 처리 프로세스

```
피드백 수집 → 분류 → 우선순위 지정 → 액션 → 검증 → 기록
```

**액션 유형:**
1. **개선**: 개념 노트 업데이트
2. **추가**: 예시/케이스 스터디 추가
3. **삭제**: Archive로 이동
4. **연기**: 다음 분기로 이연

## 📊 사용 패턴 분석

### 추적 지표

**개념별:**
- 사용 빈도 (주/월)
- 효과성 (1-5점)
- 재사용률 (%)

**Part별:**
- 가장 많이 사용된 Part
- 가장 효과적인 Part
- 가장 적게 사용된 Part

---

### 사용 통계 템플릿 (월간)

#### 2026년 1월 사용 통계

**Part별 사용 빈도:**
| Part | 사용 횟수 | 비율 | 전월 대비 |
|------|----------|------|----------|
| Part 1 | 15회 | 30% | +5% |
| Part 2 | 20회 | 40% | +10% |
| Part 3 | 10회 | 20% | 0% |
| Part 4 | 5회 | 10% | -5% |
| Part 5 | 0회 | 0% | 0% |
| Part 6 | 0회 | 0% | 0% |

**개념별 Top 5:**
1. SVD/PCA - 8회
2. 그래프 이론 - 7회
3. 베이즈 정리 - 6회
4. 내쉬 균형 - 5회
5. Null Space - 4회

**인사이트:**
- Part 2가 가장 활발히 사용됨 (40%)
- Part 5-6은 사용 없음 → 개선 필요
- SVD/PCA가 가장 인기

**액션:**
- Part 5-6 실용성 강화
- Part 2 케이스 스터디 추가

---

### 효과성 평가 (분기별)

**평가 기준:**
1. **문제 해결**: 실제 문제 해결에 기여했는가?
2. **이해 용이성**: 사용자가 쉽게 이해했는가?
3. **재사용 의향**: 다시 사용하고 싶은가?
4. **ROI**: 투입 대비 가치가 있었는가?
5. **추천 의향**: 타인에게 추천하겠는가?

각 항목 1-5점, 평균 계산

## 🔧 일반적인 문제와 해결

### 문제 1: 링크 깨짐

**증상:** 클릭 시 "파일 없음"

**원인:**
- 파일명 변경
- 파일 이동
- 대소문자 오류

**해결:**
```bash
# Obsidian에서 파일명 변경 시 자동 링크 업데이트
# 수동 변경 시:
grep -r "깨진_링크" 3-Resources/
# 해당 파일들 수동 수정
```

---

### 문제 2: 개념이 이해 안 됨

**증상:** 사용자 피드백 "너무 어려움"

**진단:**
- 불변량 1 (복잡성 단순화) 위반 가능성
- [[0_Invariants_Guide]] 재검증 필요

**해결:**
1. 예시 추가
2. 시각화 추가
3. 단계별 가이드 작성
4. 필요 시 재설계

---

### 문제 3: 사용 빈도 낮음

**증상:** 3개월 이상 사용 없음

**진단:**
- 불변량 5 (실용주의) 위반 가능성
- 실제 문제와 연결 부족

**해결:**
1. 실전 사례 추가
2. Quick Start에 포함
3. 워크샵에서 홍보
4. 개선 안 되면 Archive
