# 문제 진단 플로우차트 (Problem Diagnosis Flowchart)

**↑ [[0_BrainTwin_Master_Index|BrainTwin 마스터 인덱스로 돌아가기]]**

---

## 개요

어떤 문제든 적절한 BrainTwin Part와 도구로 연결하는 의사결정 트리. 5분 이내에 어디서 시작할지 결정할 수 있다.

**불변량 준수**:
- ✅ **복잡성 단순화**: 42개 개념 → 3-4개 질문으로 도구 선택
- ✅ **실용주의**: 즉시 적용 가능한 첫 단계 제시
- ✅ **구조적 사고**: 문제 유형별 구조화된 접근

---

## 빠른 진단 (30초)

### 1단계: 문제의 핵심은?

```
나의 문제는 주로...

[사람/조직] → Part 1
[프로세스/운영] → Part 2
[전략/경쟁] → Part 3
[의사결정/데이터] → Part 4
[혁신/창의성] → Part 5
[미래 기술] → Part 6
```

---

## 상세 진단 트리

### 분기 1: 문제의 주체

```
Q1: 문제가 발생하는 곳은?

A. 사람/팀/조직 구조
   → Part 1: 조직·인력
   → Q1-A로 이동

B. 업무 프로세스/운영
   → Part 2: 프로세스·운영
   → Q1-B로 이동

C. 시장/경쟁/전략
   → Part 3: 전략·경쟁
   → Q1-C로 이동

D. 의사결정/데이터 분석
   → Part 4: 의사결정·데이터
   → Q1-D로 이동

E. 제품/서비스 혁신
   → Part 5: 혁신·창의성
   → Q1-E로 이동

F. AI/양자/ESG 기술
   → Part 6: 미래 경영수학
   → Q1-F로 이동
```

## Part 1: 조직·인력 (Q1-A)

```
Q1-A: 조직 문제의 증상은?

A1. 특정 사람/팀이 병목
    → [[Part1_조직_인력/그래프 이론과 중심성|그래프 이론과 중심성]]
    도구: 네트워크 매핑, Betweenness Centrality
    시간: 2-4시간
    목표: 병목 지점 시각화

A2. 중복 업무가 많음
    → [[Part1_조직_인력/Rank와 Nullity|Rank와 Nullity]]
    도구: 활동-결과 매트릭스
    시간: 2-3시간
    목표: 독립 기여 vs 중복 식별

A3. 같은 문제가 반복됨
    → [[Part1_조직_인력/군론과 대칭성|군론과 대칭성]]
    도구: 패턴 분석, 불변량 식별
    시간: 3-5시간
    목표: 구조적 원인 발견

횡단 연결:
- 병목 발견 후 → Part 2 (프로세스 최적화)
- 반복 문제 → Part 5 (TRIZ로 구조 변경)
```

## Part 2: 프로세스·운영 (Q1-B)

```
Q1-B: 프로세스 문제의 유형은?

B1. 문제가 계속 재발함
    → [[Part2_프로세스_운영/시스템 사고와 피드백 루프|시스템 사고와 피드백 루프]]
    도구: 인과 고리 다이어그램, 강화/균형 루프
    시간: 2-4시간
    목표: 피드백 구조 시각화

B2. 지표가 너무 많아 혼란
    → [[Part2_프로세스_운영/SVD와 PCA|SVD와 PCA]]
    도구: Python + NumPy, 차원 축소
    시간: 4-6시간
    목표: 30개 KPI → 2-3개 핵심 요인

B3. 어떤 활동이 불필요한지 모름
    → [[Part2_프로세스_운영/Null Space 응용|Null Space 응용]]
    도구: 활동-결과 매트릭스
    시간: 2-3시간
    목표: 결과에 기여하지 않는 활동 식별

B4. 고객 행동 예측이 필요
    → [[Part2_프로세스_운영/마르코프 체인|마르코프 체인]]
    도구: 상태 전이 행렬, LTV 계산
    시간: 4-6시간
    목표: 고객 여정 및 LTV 예측

B5. 새 데이터로 전략 업데이트
    → [[Part2_프로세스_운영/베이즈 정리|베이즈 정리]]
    도구: 베이즈 공식, Python
    시간: 3-5시간
    목표: 실시간 믿음 업데이트

횡단 연결:
- 피드백 루프 발견 후 → Part 3 (인센티브 재설계)
- KPI 압축 후 → Part 4 (데이터 기반 의사결정)
```

## Part 3: 전략·경쟁 (Q1-C)

```
Q1-C: 전략 문제의 핵심은?

C1. 경쟁사와 균형 상태
    → [[Part3_전략_경쟁/내쉬 균형|내쉬 균형]]
    도구: 게임 이론, 페이오프 매트릭스
    시간: 3-5시간
    목표: 균형 깨기 전략

C2. 플랫폼/네트워크 비즈니스
    → [[Part3_전략_경쟁/네트워크 효과와 플랫폼 경제학|네트워크 효과와 플랫폼 경제학]]
    도구: 임계 질량 계산, 성장 모델
    시간: 4-6시간
    목표: 임계 질량 달성 전략

C3. 투자 포트폴리오 구성
    → [[Part3_전략_경쟁/포트폴리오 이론|포트폴리오 이론]]
    도구: Sharpe Ratio, 효율적 프론티어
    시간: 4-8시간
    목표: 리스크-수익 최적화

C4. 성장이 갑자기 정체
    → [[Part3_전략_경쟁/Phase Transition|Phase Transition]]
    도구: 임계점 분석, S-curve
    시간: 3-5시간
    목표: 다음 임계점 식별

C5. 비합리적 의사결정 많음
    → [[Part3_전략_경쟁/행동경제학|행동경제학]]
    도구: 편향 체크리스트, Nudge 설계
    시간: 2-4시간
    목표: 편향 제거, 의사결정 구조 개선

횡단 연결:
- 내쉬 균형 → Part 1 (조직 인센티브 재설계)
- 포트폴리오 → Part 4 (최적화 이론)
- 행동경제학 → Part 4 (실험 설계)
```

## Part 4: 의사결정·데이터 (Q1-D)

```
Q1-D: 의사결정 문제의 유형은?

D1. 상관관계인지 인과관계인지 불명확
    → [[Part4_의사결정_데이터/인과 추론|인과 추론]]
    도구: 회귀 분석, RCT, DoWhy
    시간: 6-10시간
    목표: 진짜 원인 규명

D2. A/B 테스트 필요
    → [[Part4_의사결정_데이터/실험과 학습|실험과 학습]]
    도구: A/B 테스트, Multi-Armed Bandit
    시간: 2주 (실험 기간 포함)
    목표: 최적 전략 발견

D3. 불확실성이 너무 큼
    → [[Part4_의사결정_데이터/엔트로피|엔트로피]]
    도구: Shannon Entropy, 정보 가치
    시간: 3-5시간
    목표: 불확실성 정량화

D4. 자원 배분 최적화
    → [[Part4_의사결정_데이터/최적화 이론|최적화 이론]]
    도구: Linear Programming, Excel Solver
    시간: 4-8시간
    목표: 제약 하 최적 배분

D5. 시간에 따른 의사결정
    → [[Part4_의사결정_데이터/동적 계획법과 강화학습|동적 계획법과 강화학습]]
    도구: Bellman Equation, Q-Learning
    시간: 1-2주
    목표: 순차적 최적 정책

D6. 극단 리스크 관리
    → [[Part4_의사결정_데이터/극값 이론|극값 이론]]
    도구: VaR, CVaR, 극값 분포
    시간: 4-6시간
    목표: Tail Risk 정량화

횡단 연결:
- 인과 추론 → Part 2 (프로세스 개선)
- 실험과 학습 → Part 3 (전략 검증)
- 최적화 → Part 6 (양자 컴퓨팅)
```

```markdown

[ ] 횡단 연결 확인 (다른 Part 필요?)
[ ] 팀 구성 및 역할 분담
```

## 예시: 플로우차트 사용

### 문제: "프로젝트가 항상 지연된다"

**Step 1: 주체 식별**
```
프로젝트 지연 → 조직? 프로세스? 전략?
→ 업무 흐름이 문제 → Part 1 or Part 2
```

**Step 2: 증상 구체화**
```
Q: 왜 지연되는가?
A: 특정 사람이 병목
→ Part 1: 그래프 이론과 중심성
```

**Step 3: 도구 적용**
```
도구: 네트워크 매핑, Betweenness Centrality
시간: 2-4시간
목표: 병목 지점 시각화
```

**Step 4: 횡단 확인**
```
병목 발견 후:
→ Part 2: 시스템 사고 (왜 병목이 생겼나?)
→ Part 3: 내쉬 균형 (인센티브 문제?)
→ Part 4: 실험과 학습 (해결책 검증)
```
